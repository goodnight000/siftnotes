# Full Diarization Evaluation Status

Date: 2026-06-21

## Pause Note

Paused on 2026-06-22 by product decision. Do not run pyannote locally, do not
switch to a paid/cloud diarization oracle, and do not integrate full
diarization into the app UI from the current evidence.

Reason: the local pyannote path would run CPU-only in the current environment,
cloud/remote options would require uploading audio and paying API or GPU costs,
and sherpa-onnx has not passed the overlap-heavy benchmark. Resume only if one
of these conditions changes:

- explicit approval to use a cloud or remote-GPU diarization oracle on specific
  non-sensitive or approved recordings
- a better local/native diarization candidate becomes available
- product decides that best-effort local diarization is acceptable despite the
  current overlap-heavy failure

## Decision Status

No app engine should be integrated yet.

The synthetic fixture validates the mechanics of scoring and transcript
assignment only. It is not evidence that sherpa-onnx or pyannote is best for
Meetily recordings.

The public AMI checks are real DER/JER evidence. Sherpa-onnx is strong enough
on balanced clean/noisy 2-speaker clips, but fails the 4-speaker overlap-heavy
clip even after duration and speaker-count tuning. Sherpa remains the native
product candidate because it is local and app-default-safe, but it is not good
enough to ship on current corpus evidence.

Current provisional strategy remains:

| Role | Engine |
| --- | --- |
| Quality oracle | deferred; `pyannote-community-1` is not being run locally for now |
| App-default candidate | none proven; `sherpa-onnx` remains under evaluation |
| App integration | wait until real bakeoff passes gates |
